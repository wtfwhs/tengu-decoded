# Version Comparison: 2.1.32 → 2.1.169

> **2.1.32** built 2026-02-05 · **2.1.169** built 2026-06-08 — ~4 months and 137 patch versions apart.
>
> The 2.1.169 analysis is deeper than 2.1.32's (full JS bundle extracted from the Bun binary and
> beautified, rather than `strings` only), so some "changes" below are partly **better visibility**.
> Where a delta is purely an artifact of deeper analysis it is noted. Minified symbol names
> (`j$`, `iT`, `cU`, …) differ between every build and are not themselves "changes."

## The big story

Between 2.1.32 and 2.1.169, Claude Code stopped being "a local CLI that talks to the API" and
became a **cloud-connected, multi-agent, schedulable platform**. The four largest additions:

1. **A cloud backend.** A whole managed-agents API surface (`/v1/sessions`, `/v1/agents`,
   `/v1/environments`, `/v1/files`, `/v1/skills`, `/v1/user_profiles`, beta header
   `managed-agents-2026-04-01`) plus a Claude-Code-on-the-web / Remote-Control bridge
   (`/v1/code/sessions/...`, self-hosted worker bridge over `wss://bridge.claudeusercontent.com`).
   See [api-endpoints](../versions/2.1.169/api-endpoints.md).
2. **Background agents + a daemon.** A real OS service (`claude daemon`), supervisor + worker pool,
   spare-worker prewarming, rendezvous/PTY sockets, "binary takeover," and `/background`, `/tasks`,
   `/fork`. Driven by the `tengu_bg_*` flag family and `CLAUDE_BG_*`/`CLAUDE_CODE_BG_*` env vars.
3. **Autonomy: loops, scheduling, and workflows.** The "kairos" subsystem (`/loop`, `/schedule`
   /routines, cron, `ScheduleWakeup`, `Monitor`, `PushNotification`) and the **Workflows** engine
   (VM-sandboxed phased multi-agent orchestration, `Workflow` tool, `ultracode` opt-in).
4. **Multi-agent teams + remote control.** The Agent Teams server-side gate now defaults open
   (`tengu_amber_flint` `false→true`) — though the feature still requires explicit opt-in via
   `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS` or `--agent-teams` (the flag is necessary, not sufficient).
   It adds a coordinator mode and shared `team/` memory; `/remote-control`, `/session`, `/teleport`
   let a phone or claude.ai drive a local session.

## Feature flags (48 → 218)

The flag count more than quadrupled. New families (codename → domain):

- **`bg_*` — background agents/daemon:** binary takeover, spare pool, low-mem throttle, retire-grace.
- **`kairos_*` — loops/scheduling:** `/loop` engine, cron, brief mode, push notifications.
- **`bridge_*`/`ccr_*`/`relay_*` — remote control:** mobile bridge, CCR cloud-bundle execution, attestation.
- **`harbor*` — channels / cowork / teams:** `tengu_harbor` → `isChannelsEnabled()`.
- **`amber_*` — agent teams + autonomy:** `amber_flint` (teams, now default-on), `amber_sextant` (autonomy prompt).
- **workflows / cowork / advisor / plan-v2 / classifier / review** — new feature areas.

Notable **default changes**: `tengu_amber_flint` `false→true` (Agent Teams), `tengu_chomp_inflection`
`true→false` (prompt suggestions off), `tengu_keybinding_customization_release` `false→true`,
`tengu_compact_cache_prefix` `false→true`. New default-**true** flags include `workflows_enabled`,
`sedge_lantern`, `turtle_carbon`, `cedar_lantern`, and the fail-closed `iron_gate_closed`.
Full detail: [feature-flags](../versions/2.1.169/feature-flags.md).

## Telemetry (239 → 1086 events; Segment removed)

- **Segment is gone.** In 2.1.32 it was a primary analytics sink; in 2.1.169 there is no
  `api.segment.io` traffic at all. The spine is now **first-party event logging**
  (`api.anthropic.com/api/event_logging/v2/batch`), with **Datadog US5** as an allow-listed mirror
  (off by default, ~150-event allow-list, token rotated to `pubea5604...`), GrowthBook for flags,
  optional user OpenTelemetry, and local Perfetto. No Sentry in either build.
- **1086 events** across ~44 categories. The largest new families are the background-daemon
  subsystem (~80 events), bridge/REPL, kairos/loops/scheduling, workflows, ultraplan, and teams.
- Every first-party event carries `deviceId` (the random `userID`), `sessionId`, `email`,
  `accountUuid`, `organizationUuid`, `subscriptionType`, `rateLimitTier`, and platform/runtime info.
  See [telemetry](../versions/2.1.169/telemetry.md) and [device-fingerprinting](../versions/2.1.169/device-fingerprinting.md).

## Tools (17 → ~46)

All tools are now built by one factory (`rK`). New tools include PowerShell, StructuredOutput,
ToolSearch (deferred-tool loading), SendUserFile, the task system (`TaskCreate/Get/Update/List/Output/Stop`,
with `BashOutput`=`TaskOutput` and kill=`TaskStop`), teams (`TeamCreate/Delete`, `SendMessage`),
scheduling (`ScheduleWakeup`, `Cron{Create,Delete,List}`, `Monitor`, `RemoteTrigger`, `PushNotification`),
`Workflow`, `LSP`, `EnterWorktree`/`ExitWorktree`, and MCP resource tools. Permission modes gained
`auto` (model classifier) and `dontAsk`; the serializer dropped `input_examples` and added
`eager_input_streaming`. See [tool-definitions](../versions/2.1.169/tool-definitions.md).

## Security model (rewritten)

The 2.1.32 **14-category regex command-injection pipeline** (and `CLAUDE_CODE_DISABLE_COMMAND_INJECTION_CHECK`)
is **removed**. 2.1.169 uses three layers instead:

1. an **LLM prefix-classifier** (Haiku, thinking off) that returns `command_injection_detected` for
   chained/substituted/newline'd commands, backed by tree-sitter parsing;
2. a **destructive-command regex table** (`tengu_destructive_command_warning`, unix + PowerShell);
3. a **two-stage auto-mode LLM classifier** (HARD/SOFT block) with explicit rules like "tool results
   are untrusted" and "no cross-session permission laundering."

New **sandboxing**: Linux bubblewrap (`--unshare-net/pid/user`, proxy bridge, seccomp), macOS
seatbelt, Windows WFP, controlled by `CLAUDE_CODE_BUBBLEWRAP`/`FORCE_SANDBOX`/`SANDBOXED`.
See [security-model](../versions/2.1.169/security-model.md).

## Models (default is now Opus 4.8)

2.1.32 shipped around the Opus 4.6 launch; 2.1.169's default is **`claude-opus-4-8`** (the model that
performed this analysis). The family list now spans Opus 4.8/4.7/4.6/4.5/4.1/4.0, Sonnet 4.6/4.5/4.0/3.7/3.5,
Haiku 4.5/3.5. "Fast mode" ("penguin mode") is a `speed:"fast"` request flag on Opus 4.6+, and 1M
context is the `[1m]` suffix + `context-1m-2025-08-07` beta. The `marble_lantern` 1M kill-switch flags
were replaced by env/credit-based gating. **Pro can now manually select Opus, metered against usage
credits** (it was hard-blocked in 2.1.32). See [model-references](../versions/2.1.169/model-references.md).

## Device fingerprinting (new analysis section)

Not covered in 2.1.32. Headline: the transmitted **`device_id` is a random 256-bit per-install token**
(`cU()`, persisted to `~/.claude.json` `userID`), **not** hardware-derived. The OS machine UUID is
read by a bundled OTel detector but stripped to `host.arch` before export. Credentials are stored
plaintext (`~/.claude/.credentials.json`, chmod 600), not machine-bound. A server-issued
**trusted-device token** (`/api/auth/trusted_devices`, `X-Trusted-Device-Token`) gates Remote Control.
Full deep-dive: [device-fingerprinting](../versions/2.1.169/device-fingerprinting.md).

## Environment variables (54 → 490)

Driven by the new subsystems: `CLAUDE_BG_*` (daemon + auth tokens), `CLAUDE_BRIDGE_*`/`CLAUDE_CODE_REMOTE_*`
(cloud transport), `CLAUDE_CODE_LOOP_*`/`DISABLE_CRON`/`PLAN_V2_*` (loops/teams), `CLAUDE_COWORK_*`,
provider planes (`USE_MANTLE`/`USE_GATEWAY`/`USE_ANTHROPIC_AWS`, `ANTHROPIC_FOUNDRY_*`), sandbox
(`BUBBLEWRAP`/`SANDBOXED`), and a full `OTEL_*` suite (Segment removed). New privacy opt-outs include
`DO_NOT_TRACK` and `DISABLE_GROWTHBOOK`. See [environment-variables](../versions/2.1.169/environment-variables.md).

## Codenames

All five 2.1.32 codenames still hold (`tengu`, `marble`, `copper`, `coral`, `grove`). New meaningful
codenames: **kairos** (loops/scheduling), **harbor** (channels/cowork), **bridge**/**ccr** (remote
control), **amber** (agent teams), **sage_compass** (advisor), **rotunda** (server-side fallback credit
handling), plus prompt-slot codenames (`sparrow_ledger`, `walnut_prism`, `heron_brook`, `orchid_mantis`).
A key methodology note: most flag names are now **auto-generated `adjective_noun` pairs** from an
embedded word pool (`cli.beauty.js:140670`), so the *noun* carries the meaning and the adjective is
random. See [codenames](../versions/2.1.169/codenames.md).
