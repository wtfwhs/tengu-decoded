# Version Comparison: 2.1.169 → 2.1.197

> **2.1.169** built 2026-06-08 · **2.1.197** built 2026-06-29T19:08:42Z — ~3 weeks and **28 patch
> versions** apart.
>
> Both versions are **deep analyses** (full JS bundle carved from the Bun binary and beautified, not
> `strings`). Because the analysis depth is identical, **every delta below is a real product change**,
> not a visibility artifact — the caveat that qualified the 2.1.32→2.1.169 comparison does **not**
> apply here. Minified symbol names (`j$`→`it`, `d`→`q`, `rK`→`Ys`, `cU`→`vW`, …) re-mangle on every
> build and are not themselves "changes"; a per-build symbol map lives in
> [`versions/2.1.197/bundle/_facts.md`](../versions/2.1.197/bundle/_facts.md).

## The big story

Between 2.1.32 and 2.1.169 Claude Code became a cloud-connected multi-agent platform. **2.1.169 →
2.1.197 is not another reinvention — it is three weeks of consolidation and monetisation on top of that
platform.** The skeleton (daemon, worker pool, bridge, kairos loops, workflows VM, agent teams) is
structurally identical; what moved is concentrated in four places:

1. **A new top-tier model family.** **Fable 5** and **Mythos 5** ship — a "Mythos-class" tier that the
   bundled identity prompt explicitly places *above* Opus. `best` now resolves to Fable 5, the default
   `sonnet` alias jumps to **Sonnet 5**, and the entire model layer was rewritten from a hand-coded
   routing object into a **Zod-validated declarative catalog** (`pQs`) where capabilities are data, not
   predicate functions. (+5 model ids, 56→61.) See [model-references](../versions/2.1.197/model-references.md).

2. **Monetisation surfaces: credits, overage, and Frame/Design artifacts.** A new **credits/overage
   billing** flag family (`saffron_lattice`, `saffron_credits_only_tiers`, `cedar_lattice`,
   `lantern_spool`), a **fallback-credit** accounting layer (new `tengu_fallback_credit_*` events and
   the `fallback-credit-2026-06-01` beta), and a whole **Frame artifact / Claude Design** pipeline
   (new `cobalt_plinth_*` flags, `/api/frame/deploy/*`, `/v1/design/mcp`, `/design-login`,
   `user:design:*` OAuth scopes, the `Artifact`/`DesignSync` tools). This is where most of the +25
   flags, +20 endpoints, and the new model layer's "credits-only tiers" all converge.

3. **Cloud backend depth: vaults and memory stores.** The Managed-Agents SDK gained **Vaults**
   (`/v1/vaults` + credentials + `mcp_oauth_validate` — Anthropic-managed credential storage injected at
   egress) and **Memory Stores** (`/v1/memory_stores` + memories + immutable `memory_versions/redact`).
   The CCR worker proxy added `…/worker/web-search`. See [api-endpoints](../versions/2.1.197/api-endpoints.md).

4. **Hardening, not new features.** MCP/plugin guards default-on (`mcp_server_policy_bypass_exempt`,
   `mcp_startup_policy_seed`, `mcp_stateless_skip_init` flipped to true), the auto-mode fail-closed
   path was **de-flagged into unconditional code** (`tengu_iron_gate_closed` removed), and a
   re-entrancy guard was added to the telemetry sink. The security *content* (bash classifier policy,
   36-example few-shot, sensitive-key regex, auto-mode Rules 1–10) is, after diffing, **almost
   entirely byte-identical to 2.1.169** — the genuine security deltas are narrow.

The one structural headline is the **Bun v1.3.14 → v1.4.0 runtime bump** (baked into the build:
`bun_version:1.4.0` at `cli.beauty.js:299451`/`299474`, native-runtime gate asserts `1.4.0` at
`174400`). Everything else is incremental.

## Feature flags (218 → 243, +25)

The flag count grew by 25 net (**+49 new names, ~15 removed**) via the four accessors (`it`/`J7`/`QYr`/
`$U`; renamed from `j$`/`cS`/`lS`/`OG6`). The growth clusters track exactly the "big story":

- **Frame artifacts / Design (NEW family, 7):** `cobalt_plinth*`, `saffron_anchor`,
  `frame_publish_context` — base-version rendering, reader-persist store, PUT guard, direct-upload lane.
- **Credits / overage billing (NEW family, 5):** `saffron_lattice`, `saffron_credits_only_tiers`,
  `saffron_picker_dim`, `cedar_lattice`, `lantern_spool` — overage consent, credits-only subscription
  tiers, `fallback_credit_token` header injection.
- **Remote Control defaults (5):** `cobalt_harbor` (RC on at startup), `cobalt_harbor_notice`
  (default-true auto-on banner), `harbor_moth`, `neapolitan`, `silent_harbor`.
- **MCP hardening (4):** `mcp_normalize_root_combinators`, `mcp_strip_trailing_xml_tags`, and two
  **default-true** policy guards `mcp_server_policy_bypass_exempt` / `mcp_startup_policy_seed`.

**Notable default flips:** `tengu_mcp_stateless_skip_init` **`false`→`true`** (stateless MCP servers
now skip the init handshake by default, `:268422`); new default-true guards
`mcp_server_policy_bypass_exempt`, `mcp_startup_policy_seed`, `cobalt_plinth_putguard`,
`cobalt_harbor_notice`. **No 2.1.169 default-true → false regressions were found** —
`workflows_enabled`, `send_user_file`, `turtle_carbon`, `kairos_cron`, `fleetview_pr_batch`,
`amber_flint` all kept their values.

**Most consequential removal:** `tengu_iron_gate_closed` — in 2.1.169 the default-true *fail-closed*
kill switch that denied a tool call when the auto-mode classifier couldn't run. In 2.1.197 the
fail-closed behaviour still exists but is **no longer flag-gated — it is unconditional** (`:373248`).
Other removed literals: `slim_subagent_claudemd`, `auto_background_agents`, `velvet_falcon_model`,
`agent_list_attach`, `bash_allowlist_strip_all`, `cedar_inlet`, `compass_dial`, and ~7 more.

Full detail: [feature-flags](../versions/2.1.197/feature-flags.md).

## Telemetry (1086 → 1163 events; Datadog token NOT rotated)

- **Spine unchanged.** First-party event logging (`api.anthropic.com/api/event_logging/v2/batch`) via
  the renamed fire fn `q(name,payload)` (was `d`), Datadog US5 as an opt-in allow-listed mirror,
  GrowthBook for flags, optional OTel, local Perfetto. **Segment still gone, Sentry still absent.**
- **Datadog client token was NOT rotated** — still `pubea5604404508cdd34afb69e6f42a05bc`, still US5
  (contrast 2.1.32→2.1.169, which *did* rotate it). The allow-list grew **156 → 159** events; the
  forwarded-metadata key list **dropped `kairosActive`** and **added `server_reason`/`server_type`**. A
  second Datadog endpoint (`browser-intake-us5-datadoghq.com`) now serves the Chrome-bridge RUM logger.
- **+77 net events** (+97 new / −20 removed). New families: **refusal-fallback credits**
  (`tengu_fallback_credit_minted/forfeited/skipped/outcome`), **structured-output retraction**
  (`tengu_structured_output_retracted` + `_late_retraction_drop`), **Design OAuth**
  (`tengu_design_oauth_*`), **REPL/bridge inner loop**, **Mantle plane probing**, and a
  `*_missing_trust` credential-helper-trust family. Removed: the legacy `tengu_mcp_add/delete/get/list/
  start`, `tengu_team_created/deleted`, `tengu_opus48_launch_shown`, `tengu_desktop_upsell_shown`, etc.
- **NEW re-entrancy guard** (`Fgo`, `:281333`): if the metadata collector synchronously calls
  `logEvent`, the nested event is dropped rather than recursing. The master gate `Wj()` now explicitly
  treats the **gateway plane** as a distinct telemetry-disable condition.

See [telemetry](../versions/2.1.197/telemetry.md).

## Security model (content stable; narrow real deltas)

After diffing the 2.1.169 source, the security-relevant **content is almost entirely unchanged**: the
bash policy_spec (`tcm`) and its "Code Code" header typo, the full 36-example few-shot set, the
15-member shell-interpreter blocklist (byte-identical), the sensitive-key regex (byte-identical, already
had `passwd` + `auth_header`), auto-mode User-Intent Rules 1–10 (**including Rules 8 and 9 — NOT new**),
the destructive-command tables, the subagent-report-block message, and the sandbox option set all
already existed verbatim in 2.1.169. The genuine deltas are narrow:

- **Auto-mode fail-closed de-flagged** — `tengu_iron_gate_closed` removed; the deny-when-classifier-
  unavailable path is now an unconditional circuit breaker (`:373248`, `:147092`).
- **Read-before-write bypass flags now per-model keyed** — `tengu_velvet_mallet` (Write) and
  `tengu_velvet_hammer` (Edit) are now composed via `Gst(base, model)` = `` `${base}_${model}` ``, so
  the live GrowthBook keys carry a model suffix. The Write+Edit split itself already existed.
- **`allowAppleEvents`** added to the managed-settings `sandbox` allowlist Set (the gate itself existed).
- **Cross-session defense** gained a *softer* "teammate" framing variant (`:363064`) alongside the
  existing byte-identical stern "permission laundering" framing.

The bigger MCP-side hardening is in the flag defaults (above), not the classifier. See
[security-model](../versions/2.1.197/security-model.md).

## Models (the marquee change: Fable 5 / Mythos 5 / Sonnet 5)

The model layer was **rewritten** from a hand-coded routing object (`tz`) into a Zod-validated
declarative catalog (`pQs`, `hfd`/`iMt`/`WM`/`dU`/`fQs`), with `Sfd` bridging to the legacy `va` routing
object. Capabilities are now string arrays queried via `dU(id, cap)`, not bespoke predicates.

- **+5 model ids (56 → 61):** **Fable 5** (`claude-fable-5`, "Mythos-class, the most advanced GA Claude
  model"), **Mythos 5** (`claude-mythos-5`, Fable's sibling for approved orgs), **Sonnet 5**
  (`claude-sonnet-5`). (`claude-opus-4-7-fast` is doc-only; `fable-5.md` is a docs path.)
- **Alias changes:** **`best` → Fable 5**; default **`sonnet` → Sonnet 5** (was 4.6); new **`fable`** /
  **`fable[1m]`** aliases and **`ANTHROPIC_DEFAULT_FABLE_MODEL`** env family. **Default Opus stays
  4.8** — the firstParty subscription account running this analysis resolves to `claude-opus-4-8`. The
  catalog now bakes **per-provider alias defaults** (`aliases.*.per_provider`).
- **Fast mode narrowed to Opus 4.7/4.8** — Opus 4.6 dropped; new `tengu_sunset_penguin_opus47` (default
  `2026-07-25`); `CLAUDE_CODE_ENABLE_OPUS_4_7_FAST_MODE` is now **inert** (bound but never read). Fast
  pricing simplified: Opus 4.8 fast $10/$50, Opus 4.6/4.7 fast $30/$150.
- **Overload api-key fallback broadened** from legacy-Opus-only to the **whole Opus+Fable+Mythos tier**.
- **New per-model capabilities:** `lean_prompt` (short "# Harness" — Opus 4.8/Fable 5/Mythos 5),
  `rejects_disabled_thinking` (Fable 5), `fable_5_mitigations`, `mid_conv_system`; a launch-effort
  "unpin" latch; Sonnet 5 native-1M on bedrock/vertex/foundry.

See [model-references](../versions/2.1.197/model-references.md).

## Architecture (consolidation on the 2.1.169 runtime)

The distributed runtime is the **same**: gateway-first provider selector (`Mq`→`yr`), agent teams
(`_4`→`ll`, `tengu_amber_flint` still default-on), background daemon + supervisor + worker pool + binary
takeover, remote-control bridge over `/v1/sessions`, kairos loops/cron/ScheduleWakeup, and the
VM-sandboxed Workflows engine (`hardenVMIntrinsics`, eval blocked via `codeGeneration:false`).

Incremental changes worth noting:

- **Runtime bump Bun 1.3.14 → 1.4.0** (the one structural headline; baked at `:299451`/`:299474`).
- **Coordinator gate gained a clause** (`Jv()`, `:225052`): a new `$x() && !_a() && !CLAUDE_CODE_REMOTE`
  guard biases coordinator mode toward remote/cloud sessions.
- **Coordinator prompt** (`Cup()`, `:225128`) now documents **cross-session `uds:`/`bridge:` peers** and
  `subscribe_pr_activity` GitHub-PR webhooks — stitching the local daemon and the bridge into one peer
  mesh.
- New tools (`DesignSync`/`Projects`/`Artifact`/`ShareOnboardingGuide`/`SendUserFile`) wired into the
  same kairos/agent registry block.

See [architecture](../versions/2.1.197/architecture.md).

## Bottom line

2.1.197 is a **monetisation-and-consolidation release**, not a re-platforming. The platform that landed
in 2.1.169 is intact; the three weeks since added a new top-tier model family (Fable 5 / Mythos 5 /
Sonnet 5), a credits/overage/Frame-artifact commercial layer, deeper cloud storage (vaults + memory
stores), and a round of MCP/plugin and fail-closed hardening — all on a freshly bumped Bun 1.4.0
runtime. The security and telemetry *content* barely moved, and the Datadog token was **not** rotated
this cycle.
